# 썸타임 개발 템플릿

현재 세션의 개발 주제를 분석하여 표준화된 개발 문서 템플릿을 자동으로 생성합니다. 세션에서 논의한 내용을 빠르게 정리하여 팀에 공유할 수 있는 구조화된 문서로 변환합니다.

## 발동 조건

사용자가 다음 중 하나를 입력할 때 이 스킬을 활성화합니다:
- "개발 템플릿을 작성해줘"
- "개발템플릿"
- "개발 내용 정리해줘"
- "development template"
- "dev template"
- "개발 정리"

## 동작 방식

1. **세션 맥락 분석**: 현재 대화에서 개발 주제, 목표, 기능, 관련 문서 추출
2. **정보 구조화**: 추출한 정보를 표준 템플릿 섹션에 정리
3. **문서 생성**: 완성된 개발 문서를 아래 형식으로 출력

## 생성 템플릿 형식

```
📋 개발 내용
[무엇을 개발하는지 한 줄 요약]

🎯 목적
[왜 필요한지 / 해결하는 문제]

✅ 구현 사항
* [구현 항목 1]
* [구현 항목 2]
* [구현 항목 3]
* [추가 항목...]

🔗 관련 문서
* PRD: [링크 또는 문서명]
* 디자인: [링크 또는 문서명]
* [기타 관련 문서...]

📌 참고사항
[추가 고려사항, 기술적 제약, 의존성 등]
```

## 각 섹션 작성 기준

### 📋 개발 내용
- 한 문장으로 개발 주제를 명확하게 요약
- 액션 중심의 표현 (예: "환불 기능 구현", "채팅 UI 최적화")
- 명사+동사 조합으로 구체적으로 작성

### 🎯 목적
- 2-3문장으로 개발이 필요한 이유 설명
- 유저 pain point, 비즈니스 가치, 기술적 필요성 포함
- 문제 상황 → 해결 방식 순서로 작성

### ✅ 구현 사항
- 불릿 포인트로 주요 기능/사항 나열
- 최대 5개 항목 (너무 세부적이지 않게 high-level로)
- 기능별, 카테고리별로 정리 가능
- 예: API 연동, UI 컴포넌트, 데이터 모델, 자동화 로직

### 🔗 관련 문서
- PRD, 디자인 시안 등 참고할 문서 명시
- 문서명 정확히 작성 (예: `환불_서비스_도입_배경_및_방향성`)
- 미작성 문서는 "작성 예정", "미정" 등으로 표기
- 외부 링크는 URL 포함

### 📌 참고사항
- 기술적 제약사항, 외부 의존성 기술
- 팀 간 의존성이나 협력 필요사항
- 향후 개선 계획이나 위험 요소
- 추가 고려사항 자유롭게 작성

## Examples

### 예시 1: 기능 개발
**세션 내용**: "환불 기능을 구현해야 해. 48시간 채팅 안 하면 자동으로 환불 신청 버튼 표시"

**생성 결과**:
```
📋 개발 내용
48시간 이상 응답 없는 매칭에 대한 자동 환불 버튼 기능

🎯 목적
고스팅으로 인한 사용자 불만 증가 문제 해소 및 신뢰성 강화
첫 메시지를 보내지 않는 원인(심리적 부담, 어색함)을 파악하기 위한 피드백 수집

✅ 구현 사항
* 48시간 경과 + 총 메시지 5회 미만 조건 감지
* "환불 신청하기" 버튼 UI 추가
* 구글폼 기반 환불 신청 & 피드백 수집
* 이메일 자동 답변 시스템 (프로필 피드백, 대화 팁 제공)
* 수동 결제 환불 처리

🔗 관련 문서
* PRD: 환불_서비스_도입_배경_및_방향성
* 디자인: (작성 예정)

📌 참고사항
* 환불 정책: 선택적 승인 (모든 요청 수락)
* 이메일 피드백 구성: 프로필 개선 조언 + 대화 팁
* 향후: 자동 환불 처리 및 AI 코칭 기능 추가 예정
* 의존성: 이메일 발송 서비스, 결제 API
```

### 예시 2: 시장 진출 전략
**세션 내용**: "일본 시장 진출을 위한 새로운 포지셔닝이 필요해"

**생성 결과**:
```
📋 개발 내용
일본 데이팅 시장 진출을 위한 썸타임 재팬 포지셔닝 및 마케팅 전략 수립

🎯 목적
한국-일본 문화 교류 수요 증가와 국내 포화 시장 대응
지역별 학생 커뮤니티 기반 신뢰도와 문화 교류 가치를 결합한 새로운 포지셔닝

✅ 구현 사항
* "한국-일본 문화 교류 플랫폼"으로 포지셔닝
* 오사카/교토 대학가 타깃 설정
* 일본 유저 신원 검증 체계 (이메일 인증, OCR 학생증)
* 양국 문화 교류 콘텐츠 개발
* 일본 마케팅 채널 선정 (SNS, 대학 게시판)

🔗 관련 문서
* PRD: 썸타임 재팬 기획서 (작성 중)
* 브랜드: 썸타임_브랜드_개요
* 마케팅: 일본 시장 조사 리포트 (예정)
* 기술: 일본 i18n 로컬라이제이션

📌 참고사항
* 규제: 일본 데이팅 앱 신분증 인증 법제화 검토 필요
* 현지화: 앱 UI/UX, 콘텐츠 모두 일본어 번역
* 파트너십: 일본 대학 커뮤니티, 문화 교류 단체 협력 검토
* 예상 일정: 2분기~3분기 런칭 목표
```

## Guidelines

### 효과적인 템플릿 작성

**해야 할 것** ✅
- 명확하고 구체적인 내용 작성
- "무엇을", "왜", "어떻게"를 모두 포함하기
- 관련 문서는 정확한 링크 또는 문서명 제공
- 기술적 제약사항은 명확하고 구체적으로 기술
- 세션 중 논의된 핵심 내용 모두 포함

**하지 말아야 할 것** ❌
- 모호한 표현 (예: "개선", "최적화" - 구체적으로!)
- 너무 많은 항목 (최대 5개로 제한)
- 너무 세부적인 구현 (high-level 유지)
- 오래된 문서명 (항상 최신 상태 반영)
- 불필요한 기술 용어로 복잡하게 작성

### 문서 링크 처리

- **프로젝트 내 문서**: 정확한 파일명 기재 (예: `환불_서비스_도입_배경_및_방향성`)
- **외부 링크**: 완전한 URL 포함
- **미작성 문서**: "작성 예정", "미정", "진행 중" 등으로 명확히 표기
- **링크 불확실**: "확인 필요" 등으로 표기하고 향후 업데이트 계획 명시

### 정보 부족 시 처리

- 충분한 정보가 없는 섹션은 "추후 확인 필요", "논의 예정" 등으로 표기
- 템플릿 생성 후 팀과 함께 내용 보완하기
- 개발 진행 중에도 주기적으로 템플릿 업데이트하기
