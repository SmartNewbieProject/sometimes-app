## 1\. 배경 및 문제 정의

### 1.1 현재 상황

썸타임은 현재 **Freemium 2.0 모델**(~~월 ₩4,900 구독~~ + 구슬 기반 과금)을 운영 중이나, 다음과 같은 구조적 문제에 직면해 있습니다:

| 지표 | 현재 값 | 문제점 |
| -- | -- | -- |
| LTV/CAC 비율 | 0.6  | 1.0 미만으로 구조적 적자 |
| 매칭→대화 전환율 | 0.13% | 매칭 후 실제 대화로 이어지지 않음 |
| 구매 전환율 | 3% | 대학생 지불 의향 낮음 |
| 여성 유저 이탈 | 증가 추세 | 무응답으로 인한 남성 유저 불만 가중 |

### 1.2 핵심 문제 진단

**현재 과금 구조의 치명적 오류:**

1. **재매칭(탐색) 유료화의 함정**
   * "새로운 사람을 볼 때 도파민이 터지니까 돈을 받자"는 논리
   * 실제로는 카지노 입장료를 받는 것과 동일한 실수
   * **가변적 보상(Variable Reward)** 루프를 차단하여 유저 이탈 가속
2. **호감 표시 유료화의 '이중 처벌'**
   * 돈을 내고 좋아요를 보냈는데 거절당하면:
     * 거절의 고통 (Social Rejection)
     * 금전적 손실 (Financial Loss)
   * 뇌가 앱을 "돈도 잃고 기분도 상하는 곳"으로 학습
   * **처벌 학습** → 앱 삭제 회피 반응
3. **플라이휠 효과 차단**
   * 좋아요에 과금 → 유저가 신중해짐 → System 2(숙고) 개입 → 행동 중단
   * 좋아요 총량 급감 → 받는 사람 없음 → 앱 접속 이유 상실

### 1.3 과학적 근거

| 이론 | 적용 |
| -- | -- |
| **보상 예측 오류 (RPE)** | 도파민은 보상을 '받았을 때'가 아니라 '예측할 때' 분비됨. 탐색 자체가 핵심 루프 |
| **가용성 휴리스틱** | "내 주변에 매력적인 이성이 많다"를 무료로 보여줘야 앱 가치 인식 |
| **박탈 과잉 반응** | 얻는 기쁨보다 '잃는 고통'에 민감. "누가 나를 좋아했는데 볼 수 없다" → 강렬한 결제 충동 |
| **정보 공백 이론** | "누굴까?" 호기심이 전대상피질(ACC) 자극 → 불확실성 해소 동기 |

### 2.2 기능별 상세 요구사항

#### Feature 1: 탐색 무료화 (The Hook)

| 항목 | 내용 |
| -- | -- |
| **목적** | 도파민 루프 시작, 앱 체류 시간 증가 |
| **기능** | 하루 5\~10명의 추천 프로필 무료 열람 |
| **변경점** | 기존 재매칭 유료 → 기본 매칭 무료화 |
| **제한** | 일일 무료 열람 횟수 제한 (초과 시 프리미엄 유도) |

**구현 요구사항:**

* 매일 자정 무료 열람 횟수 리셋
* 남은 무료 횟수 UI 표시
* 무료 소진 시 "더 많은 인연을 만나보세요" 프리미엄 CTA

---

#### Feature 2: 호감 표시 3단계 분리

**A. 일반 호감 (Light Signal) - 무료**

| 항목 | 내용 |
| -- | -- |
| **목적** | 플라이휠 효과 유지, 호감 데이터 유동성 확보 |
| **기능** | 기본 좋아요 하루 3\~5회 무료 |
| **효과** | "누군가는 좋아요를 받아야 앱을 킨다" |

**B. 추가 호감 (Volume) - 유료**

| 항목 | 내용 |
| -- | -- |
| **목적** | 헤비 유저의 욕구 수익화 |
| **기능** | 무료 횟수 소진 후 추가 좋아요 구슬 소모 |
| **가격** | 좋아요 1회 = 구슬 2개 (약 ₩200) |
| **타겟** | "오늘은 물이 좋네?" 느끼는 적극적 유저 |

**C. 강력한 호감 (Heavy Signal) - 유료 프리미엄**

| 항목 | 내용 |
| -- | -- |
| **목적** | "이 사람은 놓치면 안 돼" 절박함 수익화 |
| **기능** | 슈퍼 라이크 (편지 + 최상단 노출) |
| **가격** | 구슬 15개 (약 ₩1,500) |
| **효과** | 상대방에게 "돈 쓸 정도로 진심" 신호 전달 |

## 3\. 상세 요구사항

### 3.1 재화 시스템 개편

**현재:**

* 구슬 (Bead): 범용 재화

**개편 후:**

| 재화 | 용도 | 획득 방법 |
| -- | -- | -- |
| **하트 (Heart)** | 일반 호감 표시 | 매일 3\~5개 무료 지급, 구매 가능 |
| **슈퍼하트 (Super Heart)** | 강력한 호감 + 편지 | 구매 전용 |
| **열쇠 (Key)** | 나를 좋아한 사람 확인, 채팅방 열기 | 구매 전용 |

**가격 정책:**

| 패키지 | 가격 | 구성 | 단가 |
| -- | -- | -- | -- |
| 스타터 | ₩3,900 | 열쇠 5개 + 하트 10개 | ₩260/개 |
| 베이직 | ₩9,900 | 열쇠 15개 + 슈퍼하트 3개 | ₩550/개 |
| 프리미엄 | ₩19,900 | 열쇠 40개 + 슈퍼하트 10개 | ₩400/개 |

### 3.2 무료 제공 정책

| 항목 | 일일 무료 | 비고 |
| -- | -- | -- |
| 프로필 열람 | 10명 | 자정 리셋 |
| 일반 좋아요 (하트) | 5회 | 자정 리셋 |
| 슈퍼하트 | 0 | 구매 전용 |
| 나를 좋아한 사람 확인 | 0 | 구매 전용 (핵심 수익) |

### 3.3 환불 정책 연계

기존 환불 시스템과의 연계:

| 상황 | 환불 정책 |
| -- | -- |
| 채팅방 오픈 후 48시간 무응답 | 열쇠 100% 환급 |
| 나를 좋아한 사람 확인 후 탈퇴 계정 | 열쇠 100% 환급 |
| 슈퍼하트 전송 후 24시간 미확인 | 슈퍼하트 50% 환급 |

```
               │
               ▼
          [편지 작성] → [전송]
               │
               ▼
          상대방 알림: "특별한 사람이 슈퍼라이크를 보냈어요!"
```
