## **🔍 단계별 상세 분석**

### **1단계: 대학 선택 (대학선택)**

**Review**

*대학선택 페이지*

#### **현재 상태**

* 상단 프로그레스 바 (약 10\~15% 진행)
* 검색창 + 대학 리스트 카드
* "내 대학교가 없나요?" 안내 박스
* "학교 인증을 통해 안전하게 이용" 문구

#### **🚨 이탈 포인트**

| **문제점** | **영향도** | **설명** |
| -- | -- | -- |
| 긴 대학 리스트 | 중간 | 스크롤 피로감, 검색 필수화 |
| 인증 예고 문구 | 높음 | "나중에 학생증 인증해야 하나?" 부담감 |
| 학교 부재 시 | 높음 | 본인 학교가 없으면 즉시 이탈 |

#### **💡 개선 제안**

1. **지역 기반 자동 필터링**: 사용자 위치 기반으로 가까운 대학 우선 표시
2. **인증 문구 완화**: "간단한 이메일 인증" 등 부담 낮추는 워딩
3. **학교 추가 요청 CTA**: 학교가 없을 때 "추가 요청" 버튼으로 이탈 방지

---

### **2단계: 클러스터 확인**

#### **현재 상태**

* "계명대학교 선택 완료!" 확인 화면
* 소속 클러스터(지역 그룹) 안내
* 매칭 가능한 학교 리스트 표시

#### **🚨 이탈 포인트**

| **문제점** | **영향도** | **설명** |
| -- | -- | -- |
| 추가 클릭 필요 | 낮음 | 정보 확인만을 위한 불필요한 단계 |
| 클러스터 개념 혼란 | 낮음 | "클러스터가 뭐지?" 의문 유발 |

#### **💡 개선 제안**

1. **단계 통합**: 대학 선택 시 바로 클러스터 정보를 카드에 표시
2. **자동 진행**: 3초 후 자동 넘김 또는 토스트 메시지로 축소

---

### **3단계: 학과/학번/학년 입력 (추가 정보 입력하기)**

**Review**

*학과 선택 페이지*

#### **현재 상태**

* 학과 검색/선택 (Bottom Sheet)
* 학번 선택 (19\~25학번)
* 학년 선택 (1\~4학년+)

#### **🚨 이탈 포인트**

| **문제점** | **영향도** | **설명** |
| -- | -- | -- |
| 학과 검색 실패 | 높음 | 특이 학과명 검색 시 미검출 |
| 학번+학년 중복 질문 | 중간 | "21학번이면 4학년인데 왜 또 물어봐?" |
| 3개 입력 필드 | 중간 | 소개팅 앱 진입 전 "업무" 느낌 |

#### **💡 개선 제안**

1. **학과 자동완성 개선**: 초성 검색, 유사어 매칭 지원
2. **학번으로 학년 자동 추론**: 학번 선택 시 학년 자동 설정 (수정 가능)
3. **선택지 간소화**: "학부생/대학원생" 정도로 단순화 고려

---

### **4단계: 인스타그램 ID 입력**

**Review**

*인스타그램 입력 페이지*

#### **현재 상태**

* 인스타그램 아이디 입력 필드
* "프로필이 영향을 미칠 수 있습니다" 안내

#### **🚨 이탈 포인트**

| **문제점** | **영향도** | **설명** |
| -- | -- | -- |
| SNS 연동 저항 | 매우 높음 | "내 인스타를 왜 알려줘야 해?" |
| 개인정보 우려 | 높음 | 낯선 서비스에 SNS 공개 거부감 |
| 인스타 미보유 | 중간 | 인스타가 없는 사용자 처리 불명확 |

#### **💡 개선 제안**

1. **선택적 입력으로 변경**: "나중에 입력하기" 옵션 제공
2. **목적 명확화**: "매칭 후 연락용으로만 사용" 명시
3. **대안 제공**: 카카오톡 ID, 전화번호 등 대체 연락 수단

---

### **5단계: 프로필 사진 업로드**

**Review**

*프로필 사진 페이지*

#### **현재 상태**

* 3개의 사진 업로드 슬롯
* **최소 1장 필수** (다음 버튼 비활성화)
* "본인 사진을 올려주세요" 안내

#### **🚨 이탈 포인트 (가장 치명적)**

| **문제점** | **영향도** | **설명** |
| -- | -- | -- |
| 필수 사진 업로드 | 🔴 치명적 | 사진 준비 안 된 사용자 100% 이탈 |
| 초기 단계 요구 | 매우 높음 | 서비스 경험 전 개인정보(얼굴) 요구 |
| 사진 셀렉션 피로 | 높음 | "어떤 사진을 올려야 하지?" 고민 |
| 모바일 웹 한계 | 높음 | 웹에서 사진 업로드 UX 불편 |

#### **💡 개선 제안**

**CAUTION**

이 단계가 가장 큰 전환율 저하 원인으로 추정됩니다.

1. **사진 업로드 후순위화**: 프로필 완성 후 또는 첫 매칭 전에 요청
2. **임시 프로필 허용**: 기본 아바타로 가입 → 매칭 시 사진 필수
3. **점진적 공개**: "사진은 매칭된 상대에게만 공개" 명시로 안심감
4. **앱 유도**: "더 쉬운 사진 업로드는 앱에서!" 앱 설치 연결

## **🎯 우선순위별 개선 권장사항**

### **즉시 개선 (High Impact, Low Effort)**

1. 인스타그램 ID를 **선택 항목**으로 변경
2. 클러스터 확인 단계 제거 또는 통합
3. 학번 선택 시 학년 자동 추론

### **중기 개선 (High Impact, Medium Effort)**

1. 프로필 사진을 **가입 후 완성** 방식으로 변경
2. 대학 검색 UX 개선 (초성 검색, 자동완성)
3. 진행률 표시 개선 (n/m 단계 명시)

### **장기 개선 (High Impact, High Effort)**

1. 전체 가입 플로우 재설계 (필수 정보 최소화)
2. 웹 → 앱 전환 유도 최적화
3. A/B 테스트 인프라 구축
