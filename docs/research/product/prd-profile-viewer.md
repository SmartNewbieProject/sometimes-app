## 1\. 기능 개요

### 1.1 한 줄 요약

**"내 프로필을 여러 번 확인한 사람이 누구인지 구슬을 사용해 확인하고, 먼저 좋아요를 보낼 수 있는 기능"**

### 1.2 배경 및 문제 정의

**현재 상황:**

* 누군가 내 프로필을 여러 번 확인하면 푸시 알림은 발송됨
* 그러나 **"누구인지"는 확인 불가** → 설렘이 행동으로 이어지지 않음
* 푸시 알림이 다른 알림에 묻혀 놓침

**핵심 문제:**

```
"관심의 시그널은 있는데, 연결의 기회가 없다"

현재 흐름:
[상대가 내 프로필 여러 번 확인] → [푸시 알림] → [???] → 끝
  개선 흐름:
[상대가 내 프로필 여러 번 확인] → [푸시 알림] → [조회자 목록에서 확인] → [좋아요 발송] → 매칭 기회 ↑
```

**사용자 인사이트:**

* "더 알아보기"를 누르고 나간 상대 = **관심은 있지만 좋아요까지는 망설이는 상태**
* 이 사용자에게 먼저 다가갈 수 있으면 **매칭 확률 상승**
* "누군가 나를 봤다"는 기록이 쌓이면 **앱 활성화 체감** ↑

## 3\. 핵심 기능 상세

### 3.1 기능 구성

```
┌─────────────────────────────────────────────────────────┐
│                    프로필 조회자 확인                      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ① 푸시 알림                                            │
│     "누군가 당신의 프로필을 3번 확인했어요 👀"              │
│                                                         │
│  ② 앱 내 조회자 목록 (탭/메뉴)                           │
│     - 블러 처리된 프로필 카드                             │
│     - 조회 횟수 표시 (예: "3회 확인")                      │
│     - 마지막 조회 시간                                    │
│                                                         │
│  ③ 구슬 사용 → 프로필 공개                               │
│     - 공개 후 좋아요/패스 선택 가능                        │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 조회자 감지 기준

| 조건 | 포함 여부 | 비고 |
| -- | -- | -- |
| "더 알아보기" 클릭 후 프로필 확인 | ✅ | 핵심 타겟 |
| 홈에서 프로필 카드 3초 이상 체류 | ✅ | 관심 시그널 |
| 같은 사용자가 24시간 내 2회 이상 확인 | ✅ | 리스트 노출 조건 |
| 이미 매칭된 상대 | ❌ | 제외 |
| 내가 패스한 상대 | ❌ | 제외 |
| 상대가 나를 패스함 | ❌ | 제외(이건 고민해 봐야…) |

### 3.3 구슬 소비 정책

| 항목 | 구슬 | 비고 |
| -- | -- | -- |
| 프로필 1명 확인 (블러 해제) | **5구슬** | 프리미엄 기능 |
| 확인 후 좋아요 발송 | **기존 좋아요 가격** | 추가 과금 없음 |

**환불 정책:**

* 프로필 확인 후 상대가 **이미 탈퇴/휴면** 상태인 경우 → **100% 환불**

## 6\. 기술 요구사항

### 6.1 데이터 모델

```
-- 프로필 조회 로그 테이블
CREATE TABLE profile_views (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    viewer_id BIGINT NOT NULL,           -- 조회한 사람
    viewed_id BIGINT NOT NULL,           -- 조회당한 사람
    view_type ENUM('detail_view', 'card_hover') NOT NULL,
    viewed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_viewed_id_time (viewed_id, viewed_at),
    INDEX idx_viewer_viewed (viewer_id, viewed_id)
);

-- 조회자 목록 집계 테이블 (성능 최적화)
CREATE TABLE profile_viewer_summary (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    viewed_id BIGINT NOT NULL,           -- 조회당한 사람
    viewer_id BIGINT NOT NULL,           -- 조회한 사람
    view_count INT DEFAULT 1,            -- 24시간 내 조회 횟수
    first_view_at TIMESTAMP,
    last_view_at TIMESTAMP,
    is_revealed BOOLEAN DEFAULT FALSE,   -- 구슬로 확인했는지
    revealed_at TIMESTAMP NULL,
    
    UNIQUE KEY uk_viewed_viewer (viewed_id, viewer_id)
);

-- 구슬 사용 로그
CREATE TABLE gem_usage_log (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    usage_type VARCHAR(50) NOT NULL,     -- 'profile_viewer_reveal'
    gem_amount INT NOT NULL,
    related_id BIGINT NULL,              -- profile_viewer_summary.id
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 6.2 API 엔드포인트

```
GET  /api/v1/profile-viewers
     - 내 프로필 조회자 목록 (블러 상태)
     - Response: { viewers: [...], total_count: N }

POST /api/v1/profile-viewers/{viewer_id}/reveal
     - 특정 조회자 프로필 공개 (구슬 소비)
     - Request: { gem_amount: 5 }
     - Response: { success: true, profile: {...} }

GET  /api/v1/profile-viewers/count
     - 미확인 조회자 수 (뱃지용)
     - Response: { unread_count: N }
```
