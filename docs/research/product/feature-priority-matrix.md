보고서(KS‑QFD) 기준으로만 보면, “썸타임에서 더 개발(강화)해야 할 것 / 없어도 되는 것(후순위) / 잠재 욕구(매력요인)”이 꽤 명확하게 나옵니다. 특히 **Kano 분석표(12p)** 와 **개선포인트 우선순위(13p, 16p, 22p)** 가 그대로 “개발 우선순위”로 읽힙니다.

---

## 1) 이 분석이 말하는 “고객이 진짜로 원하는 것” 구조

### A. 잠재욕구(= 있으면 만족이 크게 올라가는데, 없다고 바로 욕먹지는 않는 것)

Kano에서 **‘매력’** 으로 분류된 항목(12p)이 여기에 해당합니다. 이게 곧 **차별화 기능(= 강화하면 ‘와’가 나오는 영역)** 입니다.

* **주변 사람에게 이용 사실이 알려질 걱정이 없음** (매력, 만족계수 1.00) → **강력한 익명/노출제어**가 “만족 증폭기” (12p, 16p, 18p, 22p)
* **실제로 만날 수 있는 거리·조건 매칭** (매력) → 단순 30km가 아니라 **생활권/이동패턴 + 시간대 반영 + Real‑meet Score** (12p, 16p, 18p, 22p)
* **매칭 전에 검증된 결과를 알고 싶음** (매력) → **사전 궁합/성공확률 예측의 ‘사전 표시’** (12p, 16p, 18p, 22p)
* **신뢰 가능한 실제 대학생만** (매력) → **재학 주기 검증 + 이상행동 탐지 + Profile TrustScore** (12p, 16p, 18p, 22p)

> 정리: 이 4개가 “썸타임이 글로벌 앱과 다르게 보이는” 핵심 잠재욕구입니다.

### B. 기본기(= 없으면 바로 불만/이탈로 이어지는 것)

Kano에서 **‘일원’(성능요인)**, **‘당연’(기본요건)** 으로 분류된 항목이 여기에 해당합니다(12p).
이건 “강화하면 만족도 상승”도 있지만, 더 중요한 건 **미달 시 이탈/불신/환불/리뷰 악화**로 연결되는 영역입니다.

* (당연) **왜 잘 맞는지 ‘이유’**: 없으면 불만 크고, 있어도 당연하게 여김 → **설명 가능한 매칭(XAI)** 는 “차별+기본기” 성격 (12p, 16p, 18p, 22p)
* (일원) **첫 만남 실패 최소화 / 스캠 걱정 없음 / 개인정보 보호 / 사진-실물 괴리 최소화 / 무료 체험 후 결정** 등 (12p)

## 2) 그래서 “더 개발해야 하는 기능 / 없어도 되는 기능”은 무엇인가

### 더 개발(강화)해야 하는 기능 = 보고서가 찍어준 Top 5

보고서의 경쟁적 우위요인 평가와 개선 기능 리스트가 **Top 5 개선 우선순위**를 사실상 확정합니다(13p, 16p, 22p).

1. **현실성 중심 매칭 고도화 (Real‑meet Score)**

* 생활권/이동패턴 기반 거리 모델, 시간대/이동가능성(수업/근무) 반영 (16p)
* 목표효과: “헛매칭” 감소, 첫 만남 성사율↑ (16p)

2. **사전 궁합/성공확률 ‘사전 표시’**

* 대화 데이터 기반 예측 + 초기 대화 시뮬레이션 기반 적합도 (16p)
* 목표효과: 실패 사전차단율↑, 매칭 수락률↑ (16p)

3. **대학생 신뢰 강화 (재학 주기 검증 + 이상계정 탐지 + TrustScore)**

* 1회 이메일 인증을 넘어 주기 검증 + 행동 기반 탐지 (16p)

4. **익명/노출 제어를 “옵션이 아니라 강점”으로 강화**

* 연락처·SNS 연동 차단 설계, 매칭 실패 이력 비가시화 (16p)
* 목표효과: 심리적 부담↓ → 이탈↓, 재사용률↑ (16p)

5. **설명 가능한 매칭(XAI) 리포트 강화**

* 공통점·보완점·주의 포인트 시각화, 점수 근거 투명 공개 (16p, 22p)

### “없어도 된다(= 후순위로 밀어도 된다)”로 분류되는 것

Kano에서 **‘무관심’** 으로 나온 항목은 “지금 리소스 태우면 ROI가 낮을 확률이 큰 영역”입니다(12p).

* **시간 낭비 없이 효율적으로 사람을 만나고 싶음** (무관심, 12p)
  → 해석: “효율” 자체는 결과이고, 사용자는 그걸 *별도 기능*으로 인식하지 않을 수 있음.
  → 그래서 **효율을 직접 파는 기능(예: 빠른 스와이프, 매칭 수 늘리기)** 보다는, 위의 P0들이 결과로 효율을 만들어내는 구조가 맞음(21p의 “매칭 수 경쟁 말고 불안 제거 기능 집중”과도 정합).
* **서비스 이용이 자연스럽고 긍정적으로 인식되길 원함** (무관심, 12p)
  → 이건 제품 기능으로 해결하기 어렵고(사회적 낙인/이미지), 초기에는 **익명/노출제어**로 우회 해결하는 게 더 직접적(16p).

추가로, 보고서가 강조하듯(21p) **“기능 수/화려함으로 경쟁”** 하는 영역은 의도적으로 버리는 게 맞습니다. (예: 노출량 늘리기/무한 스와이프/화려한 소셜 기능 등은 ‘성공확률↑’과 직접 연결이 약하면 후순위)

## Recommendation

1. **P0(가장 먼저)**: 보고서의 Top 5 개선 우선순위를 그대로 “로드맵의 축”으로 두는 게 합리적입니다.

* Real‑meet Score(현실성)
* 사전 검증/성공확률 사전표시
* 대학생 신뢰 강화(주기 검증/이상탐지/TrustScore)
* 익명/노출제어 강화(연락처·SNS 차단, 실패이력 비가시화 포함)
* XAI 매칭 리포트(설명 가능성)

2. **P1(동시에 최소 수준은 확보)**: 일원/당연 항목 중 **불만족 계수 큰 것(실패/스캠/개인정보/거절부담)** 을 “깨지지 않게” 유지·개선.
   → 특히 **프라이버시/노출 관련은 ‘버그=사고’** 라서 품질 게이트를 강하게.
3. **P2(후순위/삭제 후보)**: ‘효율’ ‘긍정적 인식’ 같은 무관심 항목을 목표로 하는 *독립 기능 개발*은 지금은 후순위가 맞습니다.
   → 효율은 P0의 결과로 자연스럽게 달성되는지(헛매칭↓, 사전검증↑)로 검증.

## Assumptions

* 이 보고서가 말하는 “현재(As‑Is) 스펙”은 이미 어느 정도 구현되어 있고(대학 이메일 인증, 30km 매칭, 대화 로그 수집/분석, 궁합 리포트 등), 앞으로는 **고도화/정교화**가 핵심이라는 전제(6p, 18p).
* 당장 해외 확장보다는 **국내 대학생 실증시장**에서 성과/KPI를 만든 뒤 단계 확장(9p의 결론 흐름)을 우선한다고 가정.
* “리니어 이슈”는 현재 기능 개발 현황을 반영하고 있으며, 라벨링/분류를 통해 위 P0 축과 정렬할 수 있다고 가정.

## Next steps (minimum experiments)

### 1) Linear 이슈를 “QFD 축”으로 재라벨링(1시간짜리 정리 작업)

Linear에서 아래 5개 라벨을 만들고, **In Progress/Backlog/Done** 이슈에 붙여서 “무엇에 엔지니어링 시간이 쓰이고 있는지”를 한 눈에 보이게 하세요.

* `P0-REALMEET` (생활권/이동/시간대/Real‑meet Score)
* `P0-PRECHECK` (사전궁합/성공확률/대화시뮬레이션)
* `P0-TRUST` (재학검증/이상탐지/TrustScore/스캠방지)
* `P0-PRIVACY` (익명/노출/연락처차단/비가시화/Privacy by Design)
* `P0-XAI` (매칭 근거/공통·보완·주의 설명)

→ \*\*라벨별 “Done 대비 In Progress 비율”\*\*을 보면, 지금 조직이 어디에 시간을 쓰는지 즉시 진단됩니다.
(제가 지금 이 대화에서 Linear에 직접 접근해 목록을 가져오진 못했습니다. 접근 권한/커넥터가 연결된 상태에서 이슈 리스트(Export/스크린샷/CSV)만 주시면, 위 라벨로 자동 분류해서 “정렬도/누락/낭비”까지 바로 뽑아드릴 수 있어요.)

### 2) P0별 최소 검증 지표를 “1개씩”만 잡고 2주 실험

보고서가 말하는 성능향상 포인트(16p)를 그대로 KPI로 씁니다.

* REALMEET: **헛매칭 비율↓**, **첫 만남 성사율↑**
* PRECHECK: **매칭 수락률↑**, **대화 지속률↑**, **조기 언매치↓**
* TRUST: **스캠/가짜계정 신고율↓**, **차단/오탐률(정상차단) 관리**
* PRIVACY: **이탈률↓**, **재사용률↑**, (가장 중요) **노출사고 0**
* XAI: **매칭 납득도(설문/리액션)↑**, **유료 전환율↑**

### 3) “후순위/삭제 후보” 이슈 리스트 만들기(30분)

Linear에서 최근 생성된 이슈 중,

* **매칭 수/노출을 늘리기만 하는 기능**
* **효율을 ‘기능’으로 직접 해결하려는 기능**
* **브랜딩/커뮤니티성 기능(핵심 불안 제거와 직접 연결 약함)**
  이런 것들을 한 묶음으로 모아, P0 축에 **직접 기여하는지** 기준으로 “중단/보류” 결정을 하세요(21p의 방향성과 일치).
