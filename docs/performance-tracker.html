Sometimes App 성능 개선 트래커

실제 코드에서 확인한 성능 개선 후보를 우선순위, 측정 지표, 해결 체크리스트와 함께 한 화면에 정리했습니다. 각 카드의 상태, 담당자, 메모, 체크박스는 브라우저에 자동 저장되어 해결 추적용으로 바로 사용할 수 있습니다.

범위: 초기 진입, 홈, 커뮤니티, 채팅, 이미지, i18n
기준일: 2026-03-09
저장 방식: localStorage

우선순위 맵

효과가 크고 범용 영향이 있는 작업부터 배치

추적 플로우

각 항목 공통 운영 방식
1. 계측

TTI, JS FPS, 화면 전환 시간, 네트워크 호출 수를 먼저 수집합니다.

2. 가설

렌더링, 네트워크, 번들, 이미지 중 병목 축을 하나로 좁힙니다.

3. 수정

작게 나눠 적용하고, 홈과 채팅처럼 민감한 화면은 단계적으로 배포합니다.

4. 검증

변경 전후 숫자를 비교해 개선 폭이 없으면 되돌리거나 다른 가설로 전환합니다.

필터

상태와 영역별로 카드만 골라 보기
팁: 성능 작업은 "느린 느낌"보다 숫자로 관리하는 게 훨씬 잘 굴러갑니다. 각 카드의 측정 지표를 먼저 채우고, 체크리스트 완료율과 상태를 같이 업데이트하면 회고와 재현이 쉬워집니다.
