V3 Matching UI Ideas

복합 응답(primary + scheduled + rematching)을 활용한 10가지 UI 컨셉
10 UI Concepts V3 Composite API D-? Timer Dual View

V3 응답 구조

primary 필수

  • 현재 메인으로 보여줄 매칭
  • type: open / waiting / not-found / rematching
  • partner, endOfView, connectionId

scheduled 선택

  • 정기매칭 예정/진행 정보
  • hasMatch, status, untilNext
  • 리매칭 중에도 D-? 카운트다운 가능

rematching 선택

  • 활성 리매칭 파트너 정보
  • partner, endOfView, pendingCount
  • 정기매칭 표시 중에도 리매칭 카드 가능

Idea 09

# 패턴 구현 난이도 정보 밀도 기존 UI 변경 리매칭 발견성 핵심 V3 필드
01 D-? Timer Strip 낮음 중 최소 - scheduled.untilNext
02 Split Dual Card 중간 높음 큼 높음 primary + rematching
03 PIP Card 중간 높음 중간 높음 rematching.partner
04 Swipe Carousel 중간 중 최소 중간 primary + rematching
05 Segmented Tab 낮음 중 최소 높음 pendingCount
06 Stacked Cards 높음 중 큼 중간 pendingCount
07 Bottom Sheet Peek 낮음 높음 최소 높음 rematching.partner
08 Timeline Journey 높음 높음 매우 큼 높음 전체 필드 활용
09 Floating Chip 매우 낮음 낮음 없음 중간 rematching 존재 여부
10 Dashboard Overview 중간 매우 높음 매우 큼 높음 전체 필드 활용
