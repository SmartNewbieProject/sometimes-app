V3.1 Matching UI/UX Design

6개 시나리오 완전 커버 + 학습 비용 제로 + 최소 UI 변경
6 Scenarios Primary + Secondary Category Badges Secondary Strip Zero Learning Curve

Design Principles

1

Primary = Main View

primary는 항상 기존 IdleMatchTimer 카드 위치에 렌더. 사용자가 볼 메인 콘텐츠는 항상 같은 자리.

2

Secondary = Info Strip

secondary가 있으면 카드 위에 compact strip 표시. 탭하면 해당 매칭으로 전환. 없으면 strip 자체가 사라짐.

3

Category = Color Badge

category를 색상 배지로 시각화. 보라=정기, 주황=리매칭, 파랑=글로벌. 텍스트 대신 색으로 직감적 구분.

4

Seamless Transition

21시 전후 primary 전환 시 카드만 교체. 레이아웃은 동일하므로 사용자가 "변경"을 인지하지 않음.

UI 결정 근거

Secondary Strip (기존 10안 중 Idea 1+7 혼합): D-Timer Strip의 정보 전달력 + Bottom Sheet Peek의 접근성을 결합. 단, Bottom Sheet 대신 inline strip으로 구현하여 overlay 문제를 회피하고, 카드 위에 배치하여 시선 흐름(위→아래)을 자연스럽게 유도.

Category Badge (신규): primary가 정기/리매칭/글로벌 어디서 온 건지 0.5초 내에 인지 가능해야 함. 아이콘 대신 단순 텍스트 배지를 선택하여 해석 비용 최소화. secondary가 없을 때는 숨겨서 "단독 매칭에 불필요한 라벨" 노이즈 차단.

Card Background Color: 보라=정기/기본, 주황=리매칭, 파랑=글로벌. 색상 시스템이 category와 1:1 매핑되어 badge를 읽지 않아도 무의식적으로 출처를 구분. Container 컴포넌트의 gradient 분기만 추가하면 구현 완료.

기존 UI 변경 최소화: 6개 시나리오 중 3개(normal, waiting, pending)는 기존 UI 100% 그대로. 나머지 3개도 SecondaryStrip 1개 + CategoryBadge 1개 + gradient 색상 분기만 추가. 기존 IdleMatchTimer 컴포넌트 구조를 유지하면서 composite response만 처리.

Implementation Checklist

Phase 1 — API Migration 필수

1useLatestMatching 쿼리를 GET /v3/matching으로 변경
2응답 타입을 MatchDetailsV2Composite로 변경
3types.ts에서 type 'rematching' 제거 → 'open' + category 'rematch'으로 대체
4IdleMatchTimer에서 match → response.primary 로 바인딩
5InteractionNavigation도 primary 기준으로 작동

Phase 2 — SecondaryStrip 신규

1SecondaryStrip 컴포넌트 생성 (src/features/idle-match-timer/ui/)
2Props: secondary: MatchDetailsV2 | undefined
3secondary 없으면 null 리턴 (조건부 렌더)
4secondary.type==='open' → 미니 프로필 + 타이머 + 탭 네비게이션
5secondary.type==='waiting' → D-? 카운트다운 + 날짜 텍스트
6IdleMatchTimerContent 상단에 SecondaryStrip 렌더 추가

Phase 3 — CategoryBadge 신규

1CategoryBadge 컴포넌트 (absolute positioned)
2Props: category + hasSecondary
3hasSecondary가 false면 null 리턴
4Partner 컴포넌트에 CategoryBadge 추가 (timer-badge 오른쪽)

Phase 4 — Card Gradient 수정

1Container 컴포넌트에 category prop 추가
2category='rematch' → orange gradient
3category='global' → blue gradient
4category='scheduled' 또는 undefined → 기존 purple (기본값)
5좋아요 버튼도 category에 따라 accent 색 분기
