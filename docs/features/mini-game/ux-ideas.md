# Mini Game - UI/UX 아이디어 & 확률 분포

> 각 진입 액션별 5가지 디자인 안을 제시하고,
> **예상 참여율(Engagement)**, **완료율(Completion)**, **대화 전환율(Conversation)**을
> 확률 분포로 평가합니다.

---

## 평가 기준

| 지표 | 정의 | 측정 |
|------|------|------|
| **참여율 E** | 노출 대비 게임 시작 클릭 비율 | 클릭 / 노출 |
| **완료율 C** | 시작 대비 양쪽 모두 답변 완료 | 완료 / 시작 |
| **대화 전환율 T** | 완료 후 5분 내 텍스트 메시지 발생 | 메시지 / 완료 |
| **종합 점수** | E × C × T (전체 퍼널 전환) | - |

## 진입 액션 1: 매칭 직후 아이스브레이커

> 현재 위치: `ChatScreen` → `chatList.length < 3`일 때 `NewMatchBanner` 하단

### Idea A: 인라인 카드 (Inline Card)

`NewMatchBanner` 아래에 게임 카드가 자연스럽게 이어지는 형태.
기존 `ChatGuideBanner`를 대체.

```
┌──────────────────────────────┐
│   💌 축하해요! OO님과 매칭!   │  NewMatchBanner (기존)
├──────────────────────────────┤
│                              │
│  ┌──────────────────────────┐│
│  │ 🎮 서로 알아보기          ││
│  │                          ││
│  │ 간단한 3가지 질문으로     ││
│  │ OO님과 취향을 비교해요    ││
│  │                          ││
│  │ [시작하기]  [나중에]      ││
│  └──────────────────────────┘│
│                              │
│  💡 대화 팁: ...             │  ChatGuideBanner (기존)
└──────────────────────────────┘
```

- **장점**: 가장 자연스러움, 스크롤 없이 바로 보임, 기존 UI 흐름 유지
- **단점**: 카드 크기가 작아 임팩트 부족, 무시하기 쉬움
- **컴포넌트**: `GlowingCard` 활용, `Button(primary)` + `Button(outline)`

### Idea C: 시스템 메시지 + CTA (System Message)

채팅 메시지 흐름 속에 시스템 메시지로 녹이는 형태.
첫 번째 메시지 위에 게임 카드가 시스템 메시지처럼 표시.

```

├──────────────────────────────┤
│                              │
│      ─── 시스템 ───          │
│                              │
│  ┌──────────────────────┐    │
│  │ 💡 서로 알아보기 퀴즈  │    │
│  │                      │    │
│  │ 3개 질문에 답하면      │    │
│  │ 취향 일치율을 알 수    │    │

│  │                      │    │
│  │ [ 참여하기 ]          │    │

│  ───────────────────────     │
│                              │
```

- **단점**: 메시지가 쌓이면 위로 밀려남, 놓치기 쉬움

### Idea D: 입력창 대체 (Input Replacement)

첫 메시지 전까지 입력창 자체를 게임 시작 CTA로 대체.
"메시지를 입력하세요" 대신 게임 카드가 입력 영역에 표시.

```
┌──────────────────────────────┐
│   💌 축하해요! OO님과 매칭!   │
│                              │
│   💡 대화 팁: ...             │
│                              │
│        (빈 채팅 영역)         │
│                              │
├──────────────────────────────┤
│ ┌──────────────────────────┐ │
│ │  🎮 퀴즈로 첫 대화를      │ │
│ │     시작해볼까요?         │ │
│ │                          │ │
│ │  [퀴즈 시작] [직접 대화]  │ │
│ └──────────────────────────┘ │
└──────────────────────────────┘
```

- **장점**: 입력 의도와 직결, "뭐라고 시작하지?" 고민을 대체, 가장 맥락적
- **단점**: 직접 대화 선택 시 전환 어색, 구현 복잡도 높음
- **컴포넌트**: `ChatInput` 조건부 렌더링, `Button` 2개 횡배치

### Idea E: 매칭 카드 확장 (Match Card Extension)

`NewMatchBanner` 자체를 확장하여 게임을 통합. 매칭 축하 + 게임 시작이 하나의 카드.

```
┌──────────────────────────────┐
│  ┌──────────────────────────┐│
│  │                          ││
│  │   💌 OO님과 매칭됐어요!  ││
│  │                          ││
│  │   서로의 관심사가        ││
│  │   비슷할까요?            ││
│  │                          ││
│  │   ┌─────┐ ┌─────┐       ││
│  │   │ 🎮  │ │ 💬  │       ││
│  │   │퀴즈 │ │대화 │       ││
│  │   │시작 │ │시작 │       ││
│  │   └─────┘ └─────┘       ││
│  │                          ││
│  └──────────────────────────┘│
│                              │
│        (채팅 메시지들)        │
└──────────────────────────────┘
```

- **장점**: 하나의 UX로 매칭 축하 + 게임 자연스럽게 연결, 선택권 명확
- **단점**: 배너 크기 증가, 대화 영역 축소, 두 옵션 분산
- **컴포넌트**: `NewMatchBanner` 확장, `Pressable` 카드 2개

## 진입 액션 2: 채팅 내 게임 시작 (수동 트리거)

> 현재 위치: `ChatInput` → `[+사진] [💡AI팁] [입력창]`

### Idea A: 게임 아이콘 버튼 (Icon Button)

기존 팁 버튼 옆에 게임 아이콘 추가. 탭하면 바텀시트.

```
[+사진] [💡팁] [🎮] [      입력창      ]
                 ↓ 탭
┌──────────────────────────────┐
│  ━━━                         │
│  🎮 게임 선택                 │
│                              │
│  ⚖️ 밸런스 게임     1분 >    │
│  🤔 취향 퀴즈       2분 >    │
│  📸 오늘의 미션    하루 >    │
│                              │
│  오늘 남은 게임: 2/3         │
└──────────────────────────────┘
```

- **장점**: 기존 패턴(`💡팁` 버튼) 일관, 학습 비용 낮음, 공간 효율
- **단점**: 아이콘 작아서 발견성 낮음, 버튼 3개로 입력창 축소
- **컴포넌트**: `Pressable` + SVG 아이콘, `BottomSheetPicker` 커스텀
