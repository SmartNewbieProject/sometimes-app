검색 모드에서 뒤로가기

✓ 정상 동작
1
검색창 터치 → 검색 모드 진입
isFocused = true, trigger = true
2
뒤로가기 → 검색 모드만 해제
isFocused가 true이면 fallback 실행 없이 검색 UI만 닫음. 로그인 이동 없음.

미선택 상태 뒤로가기

✓ 수정됨
!
버그 (수정 전)
확인 팝업 없이 즉시 로그인으로 이동 → 재인증 피로
✓
수정 후 동작
isFocused=false인 상태에서 fallback 실행 → 확인 모달 표시
// university.tsx — handleBackPress const handleBackPress = () => { onBackPress(() => { showModal({ title: '회원가입을 중단하시겠습니까?', primaryButton: { text: '이동', onClick: () => router.navigate('/auth/login') }, secondaryButton: { text: '취소' } }); }); };

대학 선택 후 뒤로가기

✓ 수정됨
!
버그 (수정 전)
카드 선택 상태를 무시하고 바로 로그인 이동
✓
수정 후 동작
선택 여부와 무관하게 fallback = 확인 모달로 통일

모달에서 이동 확인

UX 개선
1
모달 확인 버튼 탭 → 로그인 이동
hideModal() 후 router.navigate('/auth/login')
2
취소 버튼 탭 → 대학 선택 유지
모달만 닫히고 화면 상태 유지됨
Android BackHandler
useEffect로 isFocused 의존성 등록.
검색 모드 진입/해제 시 핸들러가 자동 갱신됨.
