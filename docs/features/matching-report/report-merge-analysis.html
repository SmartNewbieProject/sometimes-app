장점 매트릭스 — 각 화면에서 가져올 Best Point

OnboardingResult 감성 · 시각 강점
  • 풀스크린 그라디언트 배경 — #FFFFFF→#F5F1FF→#DECEFF→#B095E0, 4-stop 부드러운 색상 전환으로 보고서 분위기 연출
  • 미호 캐릭터 이미지 헤더 — miho-mailbox.webp 70×70 원형, 브랜드 감성 진입점 제공
  • DimensionStatsSection (CircularProgress) — 원형 진행바 + 가로 바 조합, 시각적 직관성 높음
  • 완전한 narrative 섹션 셋 — StoryCard, GrowthJourney, PersonaNarrative가 항상 표시됨
  • TitleAwardSection에 ceremonyText — 칭호 수여 의식 문구로 감동 연출
WeeklyReportPage 기능 · 네비게이션 강점
  • 컴팩트 헤더 네비게이션 — 뒤로가기 버튼 + 미호 아바타 + 제목/주차 텍스트 + 날짜 배지, 명확한 컨텍스트
  • AnimatedStatRow — 36px 대형 숫자 + ▲▼ delta + 순차 슬라이드 애니메이션, 성장 체감 극대화
  • 이전 주 비교 RadarChart — dashed polygon으로 지난 주 위치 표시, 성장 변화 시각화
  • InsightCard — 차원별 definition + feedback 텍스트, legacy 사용자에게 상세 설명 제공
  • 완전한 로딩/에러/빈 상태 UI — ActivityIndicator + 에러 CTA + 빈 데이터 안내
  • useMomentAnalytics 트래킹 — reportView · chartView · keywordView 이벤트 전송
병합 시 신규 추가 통합 화면 개선 사항
  • narrative 모드에서도 delta 표시 — DimensionStatsSection에 prevScore 연동, CircularProgress 옆 ▲▼ 표기
  • narrative RadarChart에 이전 주 비교선 추가 — weekly의 dashed polygon 로직을 NarrativeRadarChart에 이식
  • 미호 캐릭터를 헤더 아래 인라인 배치 — 컴팩트 헤더 하단에 소형 캐릭터 이미지 섹션 추가
  • OnboardingResult에 애널리틱스 추가 — useMomentAnalytics 훅 이식
  • 단일 파일로 유지보수 통합 — weekly-report-page.tsx가 양쪽 타입을 모두 처리

OnboardingResult 처리 방향: 온보딩 완료 후 라우팅을 WeeklyReportPage로 통일. reportType: 'onboarding' 파라미터를 URL에 포함하여 동일 화면에서 렌더링. OnboardingResult 파일은 추후 제거하거나 WeeklyReportPage로의 래퍼로 남김.

구현 우선순위 — 변경 과제 목록

1
narrative 모드 RadarChart에 이전 주 비교선 추가
현재 NarrativeRadarChart는 이번 주 데이터만 그림. WeeklyReportPage의 RadarChart가 구현한 prevPolygon + dashed stroke 로직을 narrative-report-components.tsx의 공유 컴포넌트에 이식. props에 prevRadarData? 옵셔널 추가 후, OnboardingReport와 WeeklyReport 양쪽에서 prevScore 데이터를 넘기도록 변경.
임팩트: 높음 난이도: 중간 신규 기능
2
narrative DimensionStats에 delta(▲▼) 표시 추가
DimensionStatsSection에 prevScores?: Record<string, number> props 추가. 내부에서 currentScore - prevScore 계산 후 CircularProgress 옆 또는 가로 바 우측에 ▲+N / ▼-N 텍스트 표시. legacy AnimatedStatRow의 delta 표현 방식(초록/빨강 색상)을 그대로 참조.
임팩트: 높음 난이도: 낮음 신규 기능
3
OnboardingResult → WeeklyReportPage 라우팅 통합
온보딩 완료 후 router.replace 대상을 /moment/weekly-report?reportType=onboarding&reportData=...로 변경. WeeklyReportPage에서 reportType=onboarding 파라미터를 감지하여 CTA를 "모먼트 시작하기"로 교체. useLocalSearchParams에 reportData?: string 파싱 로직 추가. OnboardingResult는 deprecated 처리.
임팩트: 높음 난이도: 중간 리팩토링
4
미호 캐릭터 섹션 — 헤더 아래 인라인 배치
WeeklyReportPage narrative 모드 진입 시, 컴팩트 헤더 바로 아래에 미호 이미지(miho-mailbox.webp) + 완료 타이틀 텍스트 섹션 추가. 기존 OnboardingResult의 headerSection 스타일 그대로 이식. 이미지 크기 70→56px로 축소하여 컴팩트 헤더와 밀도 균형 유지.
임팩트: 중간 난이도: 낮음 신규 기능
5
useMomentAnalytics — OnboardingResult 이식
현재 OnboardingResult에 애널리틱스 트래킹이 없음. WeeklyReportPage의 useEffect 트래킹 블록(trackWeeklyReportView, trackReportChartView, trackReportKeywordView)을 이식. 라우팅 통합(과제 3) 후에는 자동으로 해결되므로 별도 작업 불필요.
임팩트: 중간 난이도: 낮음 신규 기능
6
narrative RadarChart — 이번 주/지난 주 범례 추가
WeeklyReportPage의 legacy 모드에는 범례가 있지만, narrative 모드의 NarrativeRadarChart 아래에는 없음. 과제 1(이전 주 비교선 추가) 이후, 차트 하단에 ■ 이번 주 / ─ ─ 지난 주 범례 행을 추가. chartWrapper 내부에 legend View 삽입.
임팩트: 중간 난이도: 낮음 신규 기능
7
OnboardingResult 파일 제거 (클린업)
라우팅 통합 완료 후, src/features/moment/ui/onboarding/onboarding-result.tsx 삭제. app/ 디렉토리의 온보딩 결과 라우트 파일도 WeeklyReportPage로 리다이렉트 또는 제거. 관련 import/export를 moment/ui/index.ts에서 정리.
임팩트: 낮음 (코드 정리) 난이도: 낮음 리팩토링
