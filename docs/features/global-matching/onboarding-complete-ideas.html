API 연동 구현 방안

1. 온보딩 API 분리 (핵심 변경)

현재 useGlobalRematch에서 completeOnboarding + globalRematch를 동시 호출합니다.

개선: useCompleteOnboarding 훅을 별도 생성하여 온보딩 완료를 독립 mutation으로 분리
  • OnboardingBottomSheet "완료" → completeOnboarding API 직접 호출
  • 성공 시 → 완료 피드백 UI 표시
  • useGlobalRematch에서 온보딩 로직 제거
  • status 쿼리 invalidate → onboardingCompleted: true 반영

2. useCompleteOnboarding 훅

  • mutationFn: POST /global-matching/onboarding
  • onSuccess: status 쿼리 invalidate + completeFirstMatch()
  • onError: 에러 모달 표시 (시트 유지)
  • 반환: { mutate, isPending, isSuccess, data }
Response 활용: data.profileTranslated 로 번역 상태 UI 표시, data.preferenceCount로 저장 확인

3. OnboardingBottomSheet 변경

  • "완료" 버튼 → useCompleteOnboarding.mutate() 호출
  • isPending 시 → 버튼 로딩 상태
  • isSuccess 시 → 시트 내용을 성공 UI로 전환
  • 성공 UI에서 "매칭 시작" → onComplete(ids) + 시트 닫기

4. useGlobalRematch 단순화

  • 온보딩 관련 코드 제거 (completeOnboarding 호출 삭제)
  • shouldCompleteOnboarding 분기 제거
  • 순수하게 globalRematch()만 호출
  • 에러 처리는 기존 유지
안전장치: 서버 측에서 온보딩 미완료 시 rematch 거부 가능. 프론트에서도 status.onboardingCompleted 체크 유지
